Project Idea

Overview

My project idea is a vacation planner, it allows users to plan a vacation or a trip out. Users will have the ability to set a budget, view time/distance (and possibly traffic) between locations, set the amount of people joining in the trip, write what they are bringing on the trip and create checklists. Most obviously planning what to do and for how long. Users will also be able to view previous trips and login to be used across several devices, users will be able to share the trip to other people. Users may also view and share previous trips.

Motivation

Recently, me and my family have been going out on trips spanning the entire day. Scheduling how we would spend our day has always been the most difficult part, we would write it on excel, which was inefficient and awkward to use. Among my project ideas; a puzzle game based off of Tetris involving words for fun (I ultimately decided that this wouldn't particularly solve a problem and is better suited as a personal project) and an application for companies that would be used to sort inventory (This one whilst possibly good for a learning experience, didn't seem that interesting to me), a vacation scheduling application felt the most viable for what skills I expect to learn. Yet ambitious enough that the project will provide a good challenge and learning experience. This was a problem I thought would have a market, as restrictions around the world are loosening.

Description

The application would allow users to do several things to plan out a vacation, from a single day out to an entire trip. Its' features include the ability for users to register and log in. This also means that users are able to login and view their data from separate devices. The application will be usable with both mobile phones and computers, for ease of viewing (most importantly, when on the trip) and more users. Users are also able to use the application to set locations and the order they would be travelled to. The application will give estimates on how long getting from location to location, using data from google maps, taking traffic into consideration, users are able to adjust this time if they want. Users may also make a checklist of what they want to do, which can be checked off by the user. Users will be able to view previous trips and also set the amount of people on the trip (extra details like each person's name and faces are optional, putting other people in your trip plans is also possible. Users can share future, current and previous trips with each other, the plan can be exported as a PDF upon request. Users may also set a budget, planning/estimating how much they wish to spend in specific. Users are also able to create a list of what they are bringing on the trip, the quantity, names and descriptions of these items are made by the user.

A log out feature simply allows users to log out of their account, returning to the login/home screen areas and/or in total. The presentation of the application will make use of logos/images to more intuitively convey the uses of the services many functions. Examples of logos used would be the standard door to symbolise logging out, features in menus acced via the three line symbol (Hamburger button), a cog to direct users to the settings, etc.. Colours would ideally be those that are easy to look at and viewable, most especially important on smaller devices like mobile phones. Colours would be low in brightness and contrast and fonts will be in darker colours to help the readability.

One online trip planner which I tested navigating around was inspirock. Some of the features were more difficult to navigate around/get, features like setting budgets were more difficult to find/make. The presentation of this website is well made, with a minimalist approach which is easy to read, colour is used sparingly and primarily for images. Another website I tested navigating around was wanderlog. This site had many similarities to inspirock, a more minimalist approach, however using more colours outside of images. It was missing some features present in inspirock. Features like more user customization, through colours and structure were present, which I had not considered in my own design idea.
